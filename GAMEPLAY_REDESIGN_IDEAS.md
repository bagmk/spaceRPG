## 0. TL;DR

1. **클릭이 지루한 진짜 이유는 "반복"이 아니라 "무력함"이다.** 네 `BALANCE_ANALYSIS.md`가 이미 증명함 — 0.5~20 CPS에서 총 플레이타임이 **1.5%만** 변한다. 빨리 눌러도 게임이 안 빨라진다. 코드로도 확인: `handleClick`은 `cosmicClockSec`(진짜 게이트)를 건드리지 않는다. 클릭은 손만 바쁘다.
2. **네 두 컨셉은 이미 절반이 코드에 있다.** 16 stage = 물질 진화 사다리(진공→쿼크→원자→별→블랙홀). 240여 종 entity = 모을 물질 아이템(`auto/click/crit/time/multiplier` 효과까지 보유). → **새로 만드는 게 아니라 재배선(rewire)하는 작업.**
3. **선결 조건:** 아이템/가챠를 붙이기 전에 **클릭(과 아이템)이 실제 게이트를 밀게** 만들어야 한다. 안 그러면 "더 빠른 클릭 → 그래도 안 빨라짐"이 그대로 반복된다.
4. "엔티티랩이 의미 없어질까?" → **폐기 아니라 변환.** entity가 곧 네가 모을 아이템이다. 자산 낭비 없음.

## 1. 현재 게임 평가

### 1.1 강점 (반드시 살릴 것)
- **테마·연출·교육성:** 빅뱅→블랙홀 16단계 서사, almanac(시대별 물리 해설), 다국어, 캔버스 연출, 5종 엔딩 + 멀티버스 프레스티지. 인디 클리커 평균을 훨씬 웃도는 프로덕션.
- **재활용 가능한 자산:** entity가 `quantum_fluctuation → proton → hydrogen → DNA → black_hole`로 이미 테마링되어 있고, 등급(`common~legendary`)·글리프 패밀리(`atom/star/cell/dna/black_hole`)·효과 타입까지 데이터로 존재. 네가 말한 "아이템"이 이미 있다.

### 1.2 핵심 결함: 클릭–진행 분리 (제일 중요)
게임은 stage를 넘기려면 **두 게이트**를 모두 채워야 한다 (`formulas.ts → canCondense`):

| 게이트 | 채우는 법 | 실제 체감 |
|---|---|---|
| `quanta ≥ threshold` | 클릭 + auto | **1초~1분이면 참.** 사실상 즉시 충족 |
| `cosmicClock ≥ cosmicTimeSec` | 시간 흐름(time 스킬/entity)만 | **수십 분~수 시간 대기.** 진짜 병목 |

→ click/auto/crit 업그레이드는 "이미 채운 게이트"만 강화 → **무의미.** 플레이어는 quanta를 순식간에 채우고 나면 cosmic clock이 차길 멍하니 기다린다. 이게 "지루함"의 정체다. **클릭 보상을 더 얹어도 안 풀린다 — 클릭이 진행을 못 밀기 때문.**

### 1.3 부차 결함
- **시스템 과잉:** skill tree(4트랙 + cross-node) + entity lab + cash shop + prestige + stage별 16개 mechanic. 각각이 서로의 명료성을 깎는다. 네 doc 결론: `time_priority`가 압도적이라 **나머지 트랙은 사실상 더미.**
- **후반(14·15·16) 수학적 클리어 불가** (네 doc). 별개 이슈지만 뿌리는 같다 — 시간 게이트 과의존.

## 3. 선결 과제 — 클릭을 진행에 연결 (이걸 먼저)

네 새 시스템이 의미를 가지려면 **"행동 → 진행" 회로**가 먼저 뚫려야 한다. 세 옵션:

| 옵션 | 내용 | 장점 | 단점 |
|---|---|---|---|
| **A. 클릭이 cosmic clock도 민다** | `handleClick`에서 `cosmicDelta` 소량 추가 (네 balance doc의 Action Item 3-A) | 가장 직접적. "누르면 빨라진다" 즉시 체감 | 시간=서사 연출인데 클릭으로 건너뛰면 서사 훼손 가능 |
| **B. 진행을 quanta 중심으로 되돌림** | cosmic time은 연출/배경으로 강등, threshold를 의미 있게 상향 (doc §5·§10–13) | 클릭·아이템·auto 전부가 진행에 직결 | 16단계 "우주 시간 흐름" 컨셉이 약해짐 |
| **C. 클릭 → 시간연료(time fuel) 토큰** | 콤보/크리가 `fuel` 적립, fuel로 clock 점프 | 시간 서사 유지 + 능동성 부여 | 새 자원 1개 추가 = 복잡도↑ |

**추천: B + 약한 A.** 이유 — 네가 붙일 "아이템 → 클릭 가속/크리"가 *quanta를 통해* 진행을 밀려면 quanta가 진짜 게이트여야 한다. 시간 게이트는 후반에 "수학적 클리어 불가" 문제까지 일으키므로(네 doc), 이 기회에 진행 비중을 quanta로 옮기는 게 두 마리 토끼.

> 검증은 코드 없이 `balance-simulator.html`로 먼저 가능. threshold·time-gate 비중만 슬라이더로 바꿔 "active play가 빨라지는가"를 눈으로 확인하고 방향을 정하면 된다.

## 4. 컨셉 A 발전 — "물질 진화"를 보이는 척추로

**핵심:** stage(시간축)는 그대로 두되, 그 안에 **플레이어가 직접 키우는 "물질 등급(matter tier)" 트랙**을 노출한다.

- **트리:** 진공 → 전자 → 양성자 → 원자 → 분자 → (복잡한 분자) → 세포 → 생명 → 복합생명 → 소행성 → 행성 → 별 → 블랙홀. (네 리스트 그대로, entity 글리프와 1:1 매핑됨)
- **메커니즘:** 클릭/수집이 현재 stage의 "대표 물질"을 응축해 **다음 tier로 진화**시킨다. 진화할 때마다 클릭 산출 baseline이 올라가고, 시각적으로 중앙 코어의 물질이 바뀐다 (`clusterMode`/`stageSprites` 재사용).
- **진행 연결:** 도감(아래 §5.2)을 일정 % 채우면 현재 stage의 quanta threshold를 낮추거나 clock을 점프 → "수집이 진행을 민다"가 성립.
- **체감:** 지금은 "시간이 흘러서 다음 우주 시대가 됨"인데, 바뀌면 "내가 물질을 키워서 다음 단계를 *열었다*"가 된다.

> 미니 요약: stage = 시간/서사 축(유지). 그 위에 "내가 키우는 물질 사다리"를 덧대 능동성을 부여. 신규 자산 거의 불필요 — entity 글리프·almanac·sprite 재사용.

## 5. 컨셉 B 발전 — 수집 → 융합(가챠) → 장착 루프

이게 "그냥 클릭"을 능동적 행위로 바꾸는 **핵심 엔진**이다. 4개 하위 시스템으로 쪼갠다.

### 5.1 수집 (Collect)
- 클릭/크리/collision이 확률적으로 **물질 아이템을 드랍**한다. 지금도 클릭마다 `pickParticleName`으로 파티클 이름을 뽑고 floating으로 띄우고 있다 → **여기에 "줍는다"만 추가**하면 됨.
- 등급 `common~legendary` (이미 `EntityRarity` 존재). 크리/콤보가 높으면 상위 등급 드랍 확률↑ → **크리가 처음으로 의미를 가짐.**

### 5.2 융합/가챠 (Fuse / Gacha)
- 같은/인접 tier 아이템 **N개 + quanta 소모 → 랜덤 상위 아이템.** 주기율표/진화 트리를 따라 결과가 결정 (예: 수소×3 → 헬륨, 또는 확률적 대성공 시 한 단계 더).
- **실패/대성공 분기**로 가챠 도파민 (핀볼식). 결과 등급이 높을수록 클릭 속도/크리 보너스가 큼.
- **도감(Almanac) 승격:** 지금 almanac은 교육용 읽기 전용. 여기에 "수집 칸"을 붙여 **completion 메타**로. 한 시대 도감 완성 → stage 보너스(=§4 진행 연결).

### 5.3 장착 (Equip)
네 아이디어 그대로 — **슬롯 수가 곧 성장 축.**

| 슬롯 | 해금 조건(예시) | 효과 |
|---|---|---|
| 1슬롯 | 첫 legendary 제작 | 장착 아이템의 클릭 산출 보너스 |
| 2슬롯 | 도감 25% / stage 7 | 두 아이템 합산 → 클릭 속도↑ |
| 3슬롯 | 도감 50% / 프레스티지 1회 | 합산 + **세트 보너스** 개방 |

- **세트 보너스:** 같은 시대 3개, 또는 같은 글리프 패밀리(`star`/`cell`/`dna`…) — **`EntityGlyph` 패밀리가 이미 존재**하므로 그대로 세트 키로 사용.

### 5.4 초당 산출 시각화 + 업그레이드
- "화면에 조금씩 나오는" auto 산출 → `FloatingNumber`/`ParticleField`로 이미 띄울 수 있음. auto도 **장착/도감으로 업그레이드** → auto 트랙이 entity-equip에 흡수됨.

> 미니 요약: 클릭 → 드랍 → 융합(가챠) → 장착 → 클릭 더 셈/빠름 → 더 좋은 드랍. **자기강화 루프.** 크리·콤보·등급이 전부 이 루프 안에서 처음으로 의미를 갖는다.

## 6. 그럼 기존 시스템은? (정리·통합 판단)

네가 걱정한 부분. 결론: **버리지 말고 역할 재배치.**

| 기존 | 신규와 충돌? | 처리안 |
|---|---|---|
| **skill tree (click/auto/crit/time)** | click/auto/crit이 equip와 역할 중복 | **부분 흡수.** click/auto/crit 효과 → 장착/세트/도감 보너스로 이전. **time만 별도 유지**(시간 게이트 담당). 트랙 UI 단순화 → 명료성↑ |
| **entity lab (중앙 구매)** | 그대로 중복 | **인벤토리로 승격.** "quanta로 구매" → "드랍 + 융합"으로 대체. 패널은 수집/장착 UI로 재사용 |
| **prestige (entropy)** | 병행 가능 | 프레스티지 통화에 **"도감 완성도 / 최고 tier"**를 추가하거나 대체 → 메타 성장 동기 |
| **stage mechanic 16종** | 클릭 미니게임 — 오히려 시너지 | 유지. fusion_window·galaxy_weaving 같은 타이밍 게임이 §5.1 드랍률과 연결되면 더 좋아짐 |

**원칙 — "one in, one out":** 새 루프를 넣을 때 기존 것 하나를 빼거나 흡수. 시스템 과잉(§1.3) 재발 방지.

## 7. 우선순위 / 프로토타입 순서 (싸게 검증)

각 단계가 **독립적으로 "재미 증가"를 검증** 가능하도록 쪼갬.

1. **(반나절) 행동→진행 회로부터.** 클릭/quanta가 실제 게이트를 밀도록 1줄 수준 실험 (또는 `balance-simulator.html`로만 검증). **"빨리 누르면 빨라진다"를 먼저 체감.** 이게 안 되면 나머지는 전부 무의미.
2. **(1~2일) 최소 수집→장착 루프.** entity를 "드랍 + 슬롯 1개"로 변환. 클릭 → 아이템 드랍 → 장착 → 클릭 산출↑. 한 바퀴 돌려보고 "능동성"이 생기는지 확인.
3. **(2~3일) 융합/가챠 + 슬롯 2·3 + 세트 보너스.** 가챠 도파민과 빌드 다양성 추가.
4. **(이후) 도감 완성 메타 → 프레스티지 연결 + skill tree 흡수.** 장기 동기와 시스템 정리.

## 8. 리스크 / 체크포인트
- **가챠 인플레:** 무한 증식 방지 — 이미 `MAX_SAFE_QUANTA`, `costScaling` 있음. 융합 비용을 등급별로 가파르게.
- **프레스티지 리셋 범위:** 아이템/도감이 carry되나? (메타 성장 vs 매 런 신선함) — **네 핵심 결정 포인트.**
- **모바일 성능:** 드랍/파티클 증가 → 풀링 필요. (이미 perf 작업 이력 있음 — `PERFORMANCE_ANALYSIS.md`)
- **밸런스 회귀:** 진행 게이트를 바꾸면 16단계 페이싱 전부 재검증 필요 → `balance-simulator.html` 활용.
